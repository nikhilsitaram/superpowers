# B4: Sentinel hook: update search path

**Avoid:** Don't change the sentinel consumption logic (rm + JSON output) -- only change the `find` search paths.

**Avoid:** Don't use `-maxdepth 3` with the new paths without checking that `.claude/claude-caliper/YYYY-MM-DD-topic/.design-approved` is reachable. The new path is `.claude/claude-caliper/<dir>/.design-approved` which is 3 levels deep from the `claude-caliper` directory, so `-maxdepth 2` is sufficient (date-dir + sentinel file).

## Steps

The existing test at `tests/hooks/caliper-test_permission_request.sh` (renamed by A3) creates sentinels under `docs/plans/` and `$cwd/.claude/worktrees/*/docs/plans/`. Update the test to use the new path.

**Test 1 (line 36-37), current:**

```bash

mkdir -p "$SENTINEL_DIR1"
```

**New:**

```bash

```

**Test 3 (line 55-56), current:**

Run the test -- it should fail because the hook still searches `docs/plans/`.

### Step 2: Update the sentinel hook find pattern

In `hooks/permission-request-accept-edits.sh`, replace line 16:

**Current:**

```bash
done < <(find "$cwd/docs/plans" "$cwd/.claude/worktrees"/*/docs/plans -maxdepth 3 -name .design-approved 2>/dev/null)
```

**New:**

```bash
done < <(find "$cwd/.claude/claude-caliper" "$cwd/.claude/worktrees"/*/.claude/claude-caliper -maxdepth 2 -name .design-approved 2>/dev/null)
```

The new path structure is `.claude/claude-caliper/YYYY-MM-DD-topic/.design-approved`, so from the `claude-caliper` directory it's 2 levels deep (date-dir then sentinel file). The worktree variant searches sibling worktrees.
