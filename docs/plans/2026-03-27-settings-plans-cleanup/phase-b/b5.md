# B5: Delete docs/plans/ and update CLAUDE.md

**Avoid:** Don't delete `docs/plans/` with `rm -rf` -- use `git rm -r docs/plans/` so git tracks the deletion. The 30 historical plan directories are recoverable from git history.

**Avoid:** Don't delete `docs/reviews/` -- that directory is separate and still used for codebase review reports.

**Avoid:** Don't forget to bump the version in `marketplace.json` -- this PR changes skill behavior (plan path, settings-aware prompting), so users need the update.

## Steps

### Step 1: Delete docs/plans/ directory

```bash
git rm -r docs/plans/
```

This removes 30 plan directories containing historical design docs, plan.json files, task prose, review records, and completion stubs. All recoverable from git history.

### Step 2: Update CLAUDE.md repo structure

In `CLAUDE.md`, update the repo structure section.

**Current (line 43):**

```text
docs/plans/       — Design docs and implementation plans
```

**New:**

```text
docs/reviews/     — Codebase review reports
```

Wait -- `docs/reviews/` is already implied by the `docs/` directory. Let me look at the full structure block. The current block is:

```text
skills/           — One directory per skill (SKILL.md + optional supporting files)
docs/plans/       — Design docs and implementation plans
docs/reviews/     — Codebase review reports
.claude-plugin/   — Plugin manifest and marketplace config
```

Remove the `docs/plans/` line entirely. The `docs/reviews/` line stays. The structure becomes:

```text
skills/           — One directory per skill (SKILL.md + optional supporting files)
docs/reviews/     — Codebase review reports
.claude-plugin/   — Plugin manifest and marketplace config
```

Plan artifacts now live in `.claude/claude-caliper/` which is gitignored and not part of the repo structure.

### Step 3: Bump version in marketplace.json

In `.claude-plugin/marketplace.json`, bump the version from `1.19.1` to `1.20.0` in all three plugin entries (lines 11, 33, 58). This is a minor version bump because:
- New `source` subcommand in caliper-settings
- Plan path change affects multiple skills
- Settings-aware prompting in design skill

### Step 4: Verify

```bash
test ! -d docs/plans && echo "docs/plans deleted"
grep -c 'docs/plans/' CLAUDE.md
jq -r '.plugins[].version' .claude-plugin/marketplace.json
```

First command confirms deletion. Second should output `0`. Third should show `1.20.0` for all three plugins.
