# A2: Safe-commands prefix/glob matching

**Avoid:** Don't modify the `extract_segments` or `extract_command_words_from_segment` functions -- they handle command parsing, not matching. The glob/prefix logic belongs only in the `is_safe` function and the safe_list loading section.

**Avoid:** Don't use actual shell glob expansion (`[[ $word == $pattern ]]`) -- that introduces security risk. Implement prefix matching by stripping the trailing `*` and using `[[ "$word" == "$prefix"* ]]`.

### Step 2: Implement prefix/glob matching in is_safe

Replace the `is_safe` function and the `mapfile` line that loads `safe_list` (lines 222-232 of `hooks/pretooluse-safe-commands.sh`).

**Current:**

```bash
mapfile -t safe_list < "$SAFE_FILE"

is_safe() {
  local word="$1"
  [[ -z "$word" ]] && return 0
  local entry
  for entry in "${safe_list[@]}"; do
    [[ "$word" == "$entry" ]] && return 0
  done
  return 1
}
```

**New:**

```bash
exact_list=()
prefix_list=()
while IFS= read -r line; do
  [[ -z "$line" ]] && continue
  if [[ "$line" == *'*' ]]; then
    prefix_list+=("${line%\*}")
  else
    exact_list+=("$line")
  fi
done < "$SAFE_FILE"

is_safe() {
  local word="$1"
  [[ -z "$word" ]] && return 0
  local entry
  for entry in "${exact_list[@]}"; do
    [[ "$word" == "$entry" ]] && return 0
  done
  for entry in "${prefix_list[@]}"; do
    [[ "$word" == "$entry"* ]] && return 0
  done
  return 1
}
```

At load time, entries are split into `exact_list` and `prefix_list`. Entries ending with `*` have their `*` stripped and go into `prefix_list` for prefix matching. All other entries go into `exact_list` for exact matching. Backward compatible -- entries without `*` behave identically.
