# A5: Update plan-review skill for file-set isolation validation

**Avoid:** Don't duplicate the validate-plan logic in the reviewer prompt — the structural check (exact path overlap) is handled deterministically by validate-plan --schema. The LLM reviewer adds value by catching logical overlap that isn't exact-path: two tasks that both need to modify the same module but declared different paths, or tasks that should share a file but were artificially split.

## Steps

### Step 1: Update plan-review SKILL.md

In `skills/plan-review/SKILL.md`, in the "What It Catches" section under "Structural Validation (schema check)", add a bullet:

```markdown
- File-set overlap within a phase (create, modify, test paths must be disjoint per task)
```

This documents that validate-plan --schema now covers file-set isolation.

### Step 2: Update reviewer-prompt.md Phase Checks section

In `skills/plan-review/reviewer-prompt.md`, find the "### Phase Checks (multi-phase plans only)" section. Add a new check for file-set isolation awareness. This check applies to both single-phase and multi-phase plans, so also update the section header:

Change the header to: `### Phase & Parallelism Checks`

Update the "Structural validation already verified" note to include:
`File-set overlap within phases (no two tasks in the same phase share create/modify/test paths).`

Add an LLM reviewer check:

```markdown
    **File-set isolation (single and multi-phase):**
    - Do any tasks in the same phase logically need to modify the same module? (Indicates bad decomposition even if paths are technically different)
    - Are shared utilities or config files properly assigned to one task, with other tasks only consuming them?

    - Flag: Two tasks modify different functions in the same file (should be one task or file should be split)
    - Flag: Task A creates a utility that Task B also needs to modify (should consolidate)
```

Both under 2,000 words.
