# A3: Add status consistency checks to validate-plan

**Avoid:** Don't add checks for "all tasks complete before phase marked complete" — that's a runtime check enforced by `do_update_status_phase()` already (review gate). The design doc's list includes this, but it's already handled by the review gate mechanism. Focus on the structural checks that can be validated at plan-authoring time: task ID prefix matching phase letter, and alphabetical phase ordering.

## Steps

### Step 1: Write failing tests

Create `tests/validate-plan/test_status_consistency.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
VALIDATE="$SCRIPT_DIR/../../scripts/validate-plan"
TMPDIR_TEST=$(mktemp -d)
trap 'rm -rf "$TMPDIR_TEST"' EXIT

pass() { echo "PASS: $1"; }
fail() { echo "FAIL: $1"; exit 1; }

make_plan() {
  local dir="$1" phases_json="$2"
  cat > "$dir/plan.json" <<PLAN
{"schema":1,"status":"Not Yet Started","workflow":"create-pr",
 "goal":"Test","architecture":"Test","tech_stack":"Test",
 "phases":$phases_json}
PLAN
}

D="$TMPDIR_TEST/t1"
mkdir -p "$D/phase-a"
echo "" > "$D/phase-a/completion.md"
echo "# A1: Task one" > "$D/phase-a/a1.md"
echo "# A2: Task two" > "$D/phase-a/a2.md"
make_plan "$D" '[{"letter":"A","name":"Phase A","status":"Not Started","depends_on":[],"rationale":"Only",
  "tasks":[
    {"id":"A1","name":"Task one","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"},
    {"id":"A2","name":"Task two","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"}
  ]}]'
bash "$VALIDATE" --schema "$D/plan.json" && pass "valid plan passes" || fail "valid plan should pass"

D="$TMPDIR_TEST/t2"
mkdir -p "$D/phase-a"
echo "" > "$D/phase-a/completion.md"
echo "# B1: Wrong prefix" > "$D/phase-a/b1.md"
make_plan "$D" '[{"letter":"A","name":"Phase A","status":"Not Started","depends_on":[],"rationale":"Only",
  "tasks":[
    {"id":"B1","name":"Wrong prefix","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"}
  ]}]'
bash "$VALIDATE" --schema "$D/plan.json" 2>/dev/null && fail "mismatched task ID prefix should fail" || pass "mismatched task ID prefix detected"

D="$TMPDIR_TEST/t3"
mkdir -p "$D/phase-b" "$D/phase-a"
echo "" > "$D/phase-b/completion.md"
echo "" > "$D/phase-a/completion.md"
echo "# B1: Task B" > "$D/phase-b/b1.md"
echo "# A1: Task A" > "$D/phase-a/a1.md"
make_plan "$D" '[
  {"letter":"B","name":"Phase B","status":"Not Started","depends_on":[],"rationale":"First",
   "tasks":[{"id":"B1","name":"Task B","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"}]},
  {"letter":"A","name":"Phase A","status":"Not Started","depends_on":["B"],"rationale":"Second",
   "tasks":[{"id":"A1","name":"Task A","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"}]}
]'
bash "$VALIDATE" --schema "$D/plan.json" 2>/dev/null && fail "out-of-order phases should fail" || pass "out-of-order phases detected"

D="$TMPDIR_TEST/t4"
mkdir -p "$D/phase-a" "$D/phase-b"
echo "" > "$D/phase-a/completion.md"
echo "" > "$D/phase-b/completion.md"
echo "# A1: Task A" > "$D/phase-a/a1.md"
echo "# B1: Task B" > "$D/phase-b/b1.md"
make_plan "$D" '[
  {"letter":"A","name":"Phase A","status":"Not Started","depends_on":[],"rationale":"First",
   "tasks":[{"id":"A1","name":"Task A","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"}]},
  {"letter":"B","name":"Phase B","status":"Not Started","depends_on":["A"],"rationale":"Second",
   "tasks":[{"id":"B1","name":"Task B","status":"pending","depends_on":[],"files":{"create":[],"modify":[],"test":[]},"verification":"echo ok","done_when":"Done"}]}
]'
bash "$VALIDATE" --schema "$D/plan.json" && pass "valid multi-phase passes" || fail "valid multi-phase should pass"

echo "All status consistency tests passed"
```

Run: `bash tests/validate-plan/test_status_consistency.sh` — tests 2 and 3 should fail.

### Step 2: Implement task ID prefix check

In `scripts/validate-plan` `do_schema()`, inside the per-task loop (after the task ID is extracted and validated), add:

```bash
      if [[ -n "$id" && -n "$letter" ]]; then
        local id_prefix="${id:0:1}"
        if [[ "$id_prefix" != "$letter" ]]; then
          err "ERROR: task_id_phase_mismatch: task $id has prefix '$id_prefix' but belongs to phase $letter"
        fi
      fi
```

### Step 3: Implement alphabetical phase ordering check

In `do_schema()`, after extracting each phase letter and checking for duplicates, add an alphabetical ordering check against the previous letter:

```bash
      if [[ -n "$letter" && -n "$prev_phase_letter" ]]; then
        if [[ "$letter" < "$prev_phase_letter" || "$letter" == "$prev_phase_letter" ]]; then
          err "ERROR: phase_order: phase $letter must come after $prev_phase_letter (alphabetical order required)"
        fi
      fi
      prev_phase_letter="$letter"
```

Initialize `local prev_phase_letter=""` before the phase loop.
