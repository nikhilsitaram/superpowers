# A1: Remove supervision schema from validate-plan

**Avoid:** Don't delete the test file outright without checking if other tests import fixtures from it. The supervision tests are self-contained (own fixtures in test body), so deletion or replacement is safe.

## Steps

### Step 1: Write tests for supervision field rejection

Update `tests/validate-plan/test_supervision_schema.sh`. The old tests validated that supervision configs were accepted. The new tests should verify that plans with or without a `supervision` field both pass schema validation — the field is simply ignored (not rejected), because existing plan.json files in the wild may still contain it.

Replace the test content:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
VALIDATE="$SCRIPT_DIR/../../scripts/validate-plan"
FIXTURES="$SCRIPT_DIR/fixtures"
TMPDIR_TEST=$(mktemp -d)
trap 'rm -rf "$TMPDIR_TEST"' EXIT

pass() { echo "PASS: $1"; }
fail() { echo "FAIL: $1"; exit 1; }

mkdir -p "$TMPDIR_TEST/phase-a"
echo "" > "$TMPDIR_TEST/phase-a/completion.md"
echo "# A1: Task one" > "$TMPDIR_TEST/phase-a/a1.md"

make_plan() {
  local supervision_block="${1:-}"
  cat > "$TMPDIR_TEST/plan.json" <<PLAN
{
  "schema": 1,
  "status": "Not Yet Started",
  "workflow": "create-pr",
  "goal": "Test goal",
  "architecture": "Test arch",
  "tech_stack": "Test stack",
  ${supervision_block}
  "phases": [{
    "letter": "A", "name": "Phase A", "status": "Not Started",
    "depends_on": [], "rationale": "Only phase",
    "tasks": [{
      "id": "A1", "name": "Task one", "status": "pending",
      "depends_on": [],
      "files": {"create": [], "modify": [], "test": []},
      "verification": "echo ok", "done_when": "Done"
    }]
  }]
}
PLAN
}

make_plan ""
bash "$VALIDATE" --schema "$TMPDIR_TEST/plan.json" && pass "plan without supervision passes" || fail "plan without supervision should pass"

make_plan '"supervision": {"orchestrator_poll_seconds": 60, "dispatcher_poll_seconds": 30, "max_intervention_attempts": 2},'
bash "$VALIDATE" --schema "$TMPDIR_TEST/plan.json" && pass "plan with legacy supervision field still passes (ignored)" || fail "plan with legacy supervision should pass (field ignored)"

echo "All supervision schema tests passed"
```

Run the tests — they should fail because the current validate-plan still validates supervision fields (it will error on invalid types, but these are valid, so they may pass). The key is that after we remove supervision validation, plans with the field are simply ignored.
