# A1: Update validate-plan: remove supervision, add file-set overlap, status, and dependency checks

**Avoid:** Don't check file-set overlap across phases — only within the same phase. Cross-phase overlap is valid because phases execute sequentially.

**Avoid:** Don't just check `create` arrays for overlap (that already exists via `duplicate_create_path`). The new check covers all three arrays: `create`, `modify`, and `test`, and also detects cross-array overlap (e.g., task A1 creates `foo.ts` while task A2 modifies `foo.ts`).

### Step 6: Add file-set overlap detection to do_schema()

In `do_schema()`, after the per-task loop for each phase (after the task iteration `done`, before the phase loop's `done`), add a within-phase overlap check:

```bash
    local -A phase_file_owners=()
    for ((t=0; t<task_count; t++)); do
      local tid
      tid=$(echo "$json" | jq -r ".phases[$p].tasks[$t].id // empty")
      for arr in create modify test; do
        local arr_count
        arr_count=$(echo "$json" | jq "(.phases[$p].tasks[$t].files.$arr // []) | length")
        for ((f=0; f<arr_count; f++)); do
          local fpath
          fpath=$(echo "$json" | jq -r ".phases[$p].tasks[$t].files.$arr[$f]")
          if [[ -n "${phase_file_owners[$fpath]+x}" ]]; then
            err "ERROR: fileset_overlap: file '$fpath' claimed by tasks ${phase_file_owners[$fpath]} and $tid in phase $letter"
          else
            phase_file_owners[$fpath]="$tid"
          fi
        done
      done
    done
```

This uses an associative array scoped per phase. Each file path is tracked to its owning task ID. If a second task claims the same path (from any of create/modify/test), an error is emitted.

### Step 8f: Add --check-deps flag

Add a new `do_check_deps()` function and wire it into the argument parser. Usage: `validate-plan --check-deps plan.json --task {TASK_ID}`. The function:

1. Finds the task by ID in plan.json
2. Reads its `depends_on` array
3. For each dependency task ID, looks up its status in plan.json
4. If any dependency task's status is not `complete`, exits with error
5. If all dependencies are `complete` (or `depends_on` is empty), exits 0

```bash
do_check_deps() {
  local plan_json="$1"
  local task_id="$2"

  local json
  json=$(cat "$plan_json")

  local phase_count
  phase_count=$(echo "$json" | jq '.phases | length')

  local deps_json=""
  for ((p=0; p<phase_count; p++)); do
    local task_count
    task_count=$(echo "$json" | jq ".phases[$p].tasks | length")
    for ((t=0; t<task_count; t++)); do
      local tid
      tid=$(echo "$json" | jq -r ".phases[$p].tasks[$t].id")
      if [[ "$tid" == "$task_id" ]]; then
        deps_json=$(echo "$json" | jq -c ".phases[$p].tasks[$t].depends_on // []")
        break 2
      fi
    done
  done

  if [[ -z "$deps_json" ]]; then
    echo "ERROR: task '$task_id' not found in plan" >&2
    exit 1
  fi

  local dep_count
  dep_count=$(echo "$deps_json" | jq 'length')
  if [[ "$dep_count" -eq 0 ]]; then
    exit 0
  fi

  for ((d=0; d<dep_count; d++)); do
    local dep_id
    dep_id=$(echo "$deps_json" | jq -r ".[$d]")
    local dep_status=""
    for ((p=0; p<phase_count; p++)); do
      local task_count
      task_count=$(echo "$json" | jq ".phases[$p].tasks | length")
      for ((t=0; t<task_count; t++)); do
        local tid
        tid=$(echo "$json" | jq -r ".phases[$p].tasks[$t].id")
        if [[ "$tid" == "$dep_id" ]]; then
          dep_status=$(echo "$json" | jq -r ".phases[$p].tasks[$t].status")
          break 2
        fi
      done
    done
    if [[ "$dep_status" != "complete" ]]; then
      echo "ERROR: dependency_not_complete: task $task_id depends on $dep_id which has status '$dep_status' (expected: complete)" >&2
      exit 1
    fi
  done

  exit 0
}
```

Wire into argument parser: add `--check-deps` case alongside existing `--check-review` and `--check-workflow`.
