# A1: Add execution_mode to validate-plan schema

**Avoid:** Don't add execution_mode validation inside a new function — add it alongside the existing `workflow` validation in `do_schema()`. The schema checker uses a flat sequential check pattern; a separate function would break that convention.

## Steps

### Step 1: Write failing test for missing execution_mode

Create `tests/validate-plan/test_execution_mode.sh` using the same test harness pattern as `tests/validate-plan/test_schema.sh` (assert_pass/assert_fail helpers, TMPDIR, fixtures copy).

Write these test cases:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
VALIDATE="$REPO_ROOT/scripts/validate-plan"
FIXTURES="$SCRIPT_DIR/fixtures"
PASS=0
FAIL=0

assert_pass() {
  local desc="$1"; shift
  if "$@" > /dev/null 2>&1; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc"
    ((FAIL++)) || true
  fi
}

assert_fail() {
  local desc="$1"; shift
  local expected_error="$1"; shift
  local output
  if output=$("$@" 2>&1); then
    echo "FAIL: $desc (expected failure, got success)"
    ((FAIL++)) || true
  elif echo "$output" | grep -q "$expected_error"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected_error' in output, got: $output)"
    ((FAIL++)) || true
  fi
}

TMPDIR=$(mktemp -d)
trap 'rm -rf "$TMPDIR"' EXIT

echo "Test 1: Missing execution_mode fails"
rm -rf "${TMPDIR:?}/"*
cp -r "$FIXTURES/valid-plan/"* "$TMPDIR/"
jq 'del(.execution_mode)' "$FIXTURES/valid-plan/plan.json" > "$TMPDIR/plan.json"
assert_fail "missing execution_mode" "missing_field: execution_mode" \
  "$VALIDATE" --schema "$TMPDIR/plan.json"

echo "Test 2: Valid execution_mode 'main' passes"
rm -rf "${TMPDIR:?}/"*
cp -r "$FIXTURES/valid-plan/"* "$TMPDIR/"
jq '.execution_mode = "main"' "$FIXTURES/valid-plan/plan.json" > "$TMPDIR/plan.json"
assert_pass "execution_mode main passes" \
  "$VALIDATE" --schema "$TMPDIR/plan.json"

echo "Test 3: Valid execution_mode 'subagents' passes"
rm -rf "${TMPDIR:?}/"*
cp -r "$FIXTURES/valid-plan/"* "$TMPDIR/"
jq '.execution_mode = "subagents"' "$FIXTURES/valid-plan/plan.json" > "$TMPDIR/plan.json"
assert_pass "execution_mode subagents passes" \
  "$VALIDATE" --schema "$TMPDIR/plan.json"

echo "Test 4: Valid execution_mode 'agent-teams' passes"
rm -rf "${TMPDIR:?}/"*
cp -r "$FIXTURES/valid-plan/"* "$TMPDIR/"
cp "$FIXTURES/valid-plan/plan.json" "$TMPDIR/plan.json"
assert_pass "execution_mode agent-teams passes (fixture default)" \
  "$VALIDATE" --schema "$TMPDIR/plan.json"

echo "Test 5: Invalid execution_mode 'parallel' fails"
rm -rf "${TMPDIR:?}/"*
cp -r "$FIXTURES/valid-plan/"* "$TMPDIR/"
jq '.execution_mode = "parallel"' "$FIXTURES/valid-plan/plan.json" > "$TMPDIR/plan.json"
assert_fail "invalid execution_mode parallel" "invalid_execution_mode" \
  "$VALIDATE" --schema "$TMPDIR/plan.json"

echo "Test 6: Invalid execution_mode empty string fails"
rm -rf "${TMPDIR:?}/"*
cp -r "$FIXTURES/valid-plan/"* "$TMPDIR/"
jq '.execution_mode = ""' "$FIXTURES/valid-plan/plan.json" > "$TMPDIR/plan.json"
assert_fail "empty execution_mode" "missing_field: execution_mode" \
  "$VALIDATE" --schema "$TMPDIR/plan.json"

echo ""
echo "Results: $PASS passed, $FAIL failed"
[ "$FAIL" -eq 0 ]
```

Run the test — it should fail because `execution_mode` validation doesn't exist yet:

```bash
bash tests/validate-plan/test_execution_mode.sh
```

Expect: Tests 1, 5, 6 pass vacuously (validation doesn't check the field, so no error), Tests 2, 3, 4 may pass or fail depending on fixture. The key is Test 1 should FAIL (missing field should be rejected but currently isn't).

### Step 2: Add execution_mode validation to do_schema()

In `scripts/validate-plan`, inside `do_schema()`, add execution_mode validation immediately after the `workflow` validation block (after line ~87 in the current file). Follow the exact same pattern:

```bash
  local execution_mode
  execution_mode=$(echo "$json" | jq -r '.execution_mode // empty')
  if [[ -z "$execution_mode" ]]; then
    err "ERROR: missing_field: execution_mode"
  else
    case "$execution_mode" in
      "main"|"subagents"|"agent-teams") ;;
      *) err "ERROR: invalid_execution_mode: $execution_mode" ;;
    esac
  fi
```
