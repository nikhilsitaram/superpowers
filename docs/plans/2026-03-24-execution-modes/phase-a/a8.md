# A8: Add execution_mode tests and update valid-plan fixture

**Avoid:** Don't hardcode test values — import the valid fixture and derive test cases by mutating it with jq, matching the pattern in test_schema.sh. Don't break existing tests — the fixture update must be backward-compatible with all existing test files that use it.

## Steps

### Step 1: Read existing fixture and test files

Read:
- `tests/validate-plan/fixtures/valid-plan/plan.json` — current fixture
- `tests/validate-plan/test_schema.sh` — existing schema tests that use the fixture

Understand what fields the fixture currently has and which tests depend on exact fixture content.

### Step 2: Add execution_mode to the valid-plan fixture

Update `tests/validate-plan/fixtures/valid-plan/plan.json` to include the `execution_mode` field. Add it after `workflow`:

```json
{
  "schema": 1,
  "status": "Not Yet Started",
  "workflow": "pr-create",
  "execution_mode": "agent-teams",
  "goal": "Build a sample feature",
  ...
}
```

Use `"agent-teams"` as the default fixture value since it's the most full-featured mode and exercises the most validation paths.

### Step 4: Run the execution_mode-specific tests

All 6 tests from A1 should pass with the updated fixture.
