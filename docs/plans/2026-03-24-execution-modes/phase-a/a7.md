# A7: Update draft-plan to include execution_mode in plan.json

**Avoid:** Don't add execution_mode selection logic to draft-plan — the design skill handles mode selection and writes it to plan.json after draft-plan returns (design step 12). Draft-plan just needs to include the field in the plan.json template so implementers know it exists, and the field should be present in generated plans with a placeholder that the design skill overwrites.

## Steps

### Step 1: Read the current draft-plan SKILL.md

Read `skills/draft-plan/SKILL.md`. Find the plan.json template (the JSON example showing required fields).

### Step 2: Add execution_mode to the plan.json template

In the plan.json fields example, add `execution_mode` after `workflow`:

```json
{
  "schema": 1,
  "status": "Not Yet Started",
  "workflow": "pr-create",
  "execution_mode": "agent-teams",
  "goal": "One sentence",
  ...
}
```

### Step 3: Add a brief note about execution_mode

Near the existing `workflow` field documentation (the paragraph starting "Optional: `success_criteria` array..."), add:

```text
`execution_mode` field controls how orchestrate dispatches tasks: `main` (lead implements sequentially), `subagents` (parallel Agent tool dispatch), or `agent-teams` (parallel teammates with push notifications). Set by the design skill based on user choice.
```
