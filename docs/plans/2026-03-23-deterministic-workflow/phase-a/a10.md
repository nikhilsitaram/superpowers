# A10: Run skill-eval for modified skills

**Avoid:** Don't create evals from scratch for skills that already have evals.json files. Check `~/.claude/skill-evals/<skill-name>/` first — if an evals.json exists, add new scenarios to the existing file.

**Avoid:** Don't test only the happy path. Each modified skill needs at least one scenario that tests the new enforcement behavior (review gate blocks advancement, review-summary parsing, etc.).

## Steps

### Step 1: Run skill-eval for orchestrate

Invoke the skill-eval skill targeting orchestrate. Create or extend evals.json with scenarios covering:

1. **Review loop enforcement** — Scenario where a reviewer outputs a review-summary with >5 actionable issues. Expected: orchestrate dispatches a fresh reviewer after fixes, does not advance immediately.
2. **Review gate blocks advancement** — Scenario where `--check-review` would fail (no reviews.json record). Expected: orchestrate does not call `--update-status` to mark phase complete.
3. **Workflow completeness** — Scenario where `--check-workflow` is called as final action. Expected: orchestrate runs the check and reports the result.

### Step 2: Run skill-eval for design

Invoke the skill-eval skill targeting design. Create or extend evals.json with scenarios covering:

1. **Plan-review dispatch** — Scenario where design dispatches plan-review after draft-plan returns. Expected: plan-review is dispatched (not skipped), review-summary is parsed.
2. **Plan-only workflow gate** — Scenario where workflow is plan-only. Expected: design calls `--check-workflow` as final action.

### Step 3: Run skill-eval for review-pr

Invoke the skill-eval skill targeting review-pr. Create or extend evals.json with scenarios covering:

1. **Automated workflow suppresses skip** — Scenario where review-pr is invoked with `--automated`. Expected: "Skip fixes, proceed" option is not presented.
2. **Manual workflow retains skip** — Scenario where review-pr is invoked without `--automated`. Expected: "Skip fixes, proceed" option is available.

### Step 4: Evaluate results

For each skill:
- Run the eval suite
- Compare before vs after pass rates
- If any regression: investigate the failing scenario, determine if it's a test issue or a real regression, fix accordingly

### Step 5: Commit eval files

```text
test: add skill-eval scenarios for deterministic workflow gates
```
