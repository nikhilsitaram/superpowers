# A4: Add review-summary JSON output to all 4 reviewer prompts

**Avoid:** Don't change the existing review checklist content in any reviewer prompt. The review-summary block is additive output — it summarizes findings in machine-parseable form alongside the existing human-readable output. Removing or restructuring existing output would break any agents that parse the current format.

**Avoid:** Don't use different field names or structures across the 4 prompts. The controller extracts the same `review-summary` block from all reviewer types — inconsistent schemas would require per-type parsing logic. All 4 must produce identical structure.

## Steps

### Step 1: Define the review-summary contract

All 4 reviewer prompts will include this instruction block in their Output section. The exact JSON schema:

````markdown
### Review Summary (Machine-Readable)

After the human-readable output above, emit a fenced code block with the info string `json review-summary`. This block is parsed by the controlling agent to enforce review gates — if it is missing or malformed, the review is treated as failed and a fresh reviewer is dispatched.

```json review-summary
{
  "issues_found": 7,
  "severity": { "critical": 1, "high": 2, "medium": 3, "low": 1 },
  "verdict": "pass",
  "issues": [
    { "id": 1, "severity": "critical", "category": "Cross-task inconsistency", "file": "src/api.ts:42", "problem": "Port mismatch", "fix": "Change to 3000" }
  ]
}
```

Rules for the summary block:
- `verdict`: "pass" when zero issues remain actionable, "fail" otherwise
- `issues_found`: total count (including low/informational)
- `severity`: counts per level (critical, high, medium, low)
- `issues[]`: one entry per issue with id (sequential integer), severity, category (from checklist), file (path:line or "N/A"), problem, fix
- If zero issues: `{"issues_found": 0, "severity": {"critical": 0, "high": 0, "medium": 0, "low": 0}, "verdict": "pass", "issues": []}`
- This block must be the LAST fenced code block in your response — the controller uses the last `json review-summary` block if multiple appear
````
