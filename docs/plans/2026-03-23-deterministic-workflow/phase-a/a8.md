# A8: Remove skip-fixes option from review-pr in automated workflows

**Avoid:** Don't remove "Skip fixes, proceed" entirely — it's still valid for manual invocations where the user is in control. Only suppress it when review-pr is invoked from an automated `merge-pr` workflow.

**Avoid:** Don't add a complex detection mechanism. The simplest approach: add an `--automated` flag that orchestrate passes when invoking review-pr in a merge-pr workflow. The flag suppresses the skip option.

## Steps

### Step 1: Add --automated flag to argument parsing

Open `skills/review-pr/SKILL.md`. Add a new argument to the Arguments table:

```markdown
| `--automated` / `-A` | Suppress "Skip fixes" option (used by merge-pr workflow) |
```

### Step 2: Update Step 4 to conditionally show skip option

In Step 4 (Present & Confirm), add a conditional:

Current options:
```text
- **Fix all** — actionable + suggestion items
- **Fix critical only** — actionable items
- **Skip fixes, proceed** — jump to Step 6
- **Other** — user provides custom instructions
```

Change to:

```text
Use AskUserQuestion with options:
- **Fix all** — actionable + suggestion items (excludes dismissed/false positives)
- **Fix critical only** — actionable items (bugs, security, correctness)
- **Skip fixes, proceed** — jump to Step 6 (omit this option when `--automated` is passed — in automated workflows, all actionable findings must be fixed to maintain audit trail integrity)
- **Other** — user provides custom instructions (e.g. "fix items 1, 3, 5")
```
