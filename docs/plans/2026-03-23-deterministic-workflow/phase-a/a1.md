# A1: Add --check-review flag to validate-plan

**Avoid:** Don't put reviews.json path logic in multiple places. The plan directory is derived from `$(dirname "$plan_json")` — use this consistently for locating reviews.json. Duplicating the path derivation creates drift risk when the plan directory structure changes.

**Avoid:** Don't add a new mode variable value for `--check-review`. The existing `MODE` + `UPDATE_TARGET` pattern in the argument parser works for `--update-status` and `--criteria` but `--check-review` needs its own `--type` and `--scope` arguments. Add `CHECK_REVIEW_TYPE` and `CHECK_REVIEW_SCOPE` variables alongside the existing ones.

## Steps

### Step 1: Add the `do_check_review` function

Add a new function after `do_criteria` (line 581) in `scripts/validate-plan`. This function:

1. Derives `plan_dir` from `$plan_json` the same way `do_schema` does: `plan_dir="$(cd "$(dirname "$plan_json")" && pwd)"`
2. Checks for `$plan_dir/reviews.json` — if missing, prints `ERROR: reviews.json not found in $plan_dir` to stderr, exits 1
3. Reads reviews.json with jq
4. Finds the latest record matching both `--type` and `--scope` using: `jq --arg t "$type" --arg s "$scope" '[.[] | select(.type == $t and .scope == $s)] | last'`
5. If no matching record found: `ERROR: no review record found for type=$type scope=$scope` to stderr, exit 1
6. Checks `verdict == "pass"` and `remaining == 0` (the `.remaining` field, defaulting to 0 if absent via `// 0`)
7. If both conditions met: exit 0
8. Otherwise: `ERROR: review failed — verdict=$verdict remaining=$remaining` to stderr, exit 1

```bash
do_check_review() {
  local plan_json="$1" review_type="$2" review_scope="$3"
  local plan_dir
  plan_dir="$(cd "$(dirname "$plan_json")" && pwd)"

  local reviews_file="$plan_dir/reviews.json"
  if [[ ! -f "$reviews_file" ]]; then
    echo "ERROR: reviews.json not found in $plan_dir" >&2
    exit 1
  fi

  local record
  record=$(jq --arg t "$review_type" --arg s "$review_scope" \
    '[.[] | select(.type == $t and .scope == $s)] | last' "$reviews_file")

  if [[ "$record" == "null" || -z "$record" ]]; then
    echo "ERROR: no review record found for type=$review_type scope=$review_scope" >&2
    exit 1
  fi

  local verdict remaining
  verdict=$(echo "$record" | jq -r '.verdict // "missing"')
  remaining=$(echo "$record" | jq -r '.remaining // 0')

  if [[ "$verdict" == "pass" && "$remaining" -eq 0 ]]; then
    exit 0
  fi

  echo "ERROR: review gate failed — type=$review_type scope=$review_scope verdict=$verdict remaining=$remaining" >&2
  exit 1
}
```

### Step 2: Add argument parsing for --check-review


Add a new case in the `while` loop for `--check-review`:

```bash
--check-review)
  MODE="check-review"
  shift
  if [[ $# -gt 0 ]]; then
    PLAN_JSON="$1"
    shift
  fi
  ;;
--type)
  shift
  if [[ $# -gt 0 ]]; then
    CHECK_REVIEW_TYPE="$1"
    shift
  fi
  ;;
--scope)
  shift
  if [[ $# -gt 0 ]]; then
    CHECK_REVIEW_SCOPE="$1"
    shift
  fi
  ;;
```

Add the dispatch case at the bottom:

```bash
check-review)
  if [[ -z "$CHECK_REVIEW_TYPE" || -z "$CHECK_REVIEW_SCOPE" ]]; then
    echo "ERROR: --check-review requires --type <TYPE> and --scope <SCOPE>" >&2
    exit 2
  fi
  do_check_review "$PLAN_JSON" "$CHECK_REVIEW_TYPE" "$CHECK_REVIEW_SCOPE"
  ;;
```
