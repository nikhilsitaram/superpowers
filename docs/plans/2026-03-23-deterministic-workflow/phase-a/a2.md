# A2: Gate --update-status on review records

**Avoid:** Don't duplicate the reviews.json reading logic from `do_check_review`. Call `do_check_review` internally from the gating code, or extract a shared helper. The gate needs the same "find latest matching record, check verdict+remaining" logic — reimplementing it doubles the maintenance surface.

**Avoid:** Don't gate task-level `--update-status`. The design doc says "Task-review records are for audit trail only — no gate checks them." Only phase and plan status updates are gated.

## Steps

### Step 1: Extract a reusable review check helper

Refactor `do_check_review` (from A1) to separate the "check and return result" logic from the "exit with error" logic. Add a helper function `check_review_record` that returns 0/1 without exiting, so both `do_check_review` and the gates can use it:

```bash
check_review_record() {
  local plan_dir="$1" review_type="$2" review_scope="$3"

  local reviews_file="$plan_dir/reviews.json"
  if [[ ! -f "$reviews_file" ]]; then
    echo "reviews.json not found in $plan_dir"
    return 1
  fi

  local record
  record=$(jq --arg t "$review_type" --arg s "$review_scope" \
    '[.[] | select(.type == $t and .scope == $s)] | last' "$reviews_file")

  if [[ "$record" == "null" || -z "$record" ]]; then
    echo "no review record for type=$review_type scope=$review_scope"
    return 1
  fi

  local verdict remaining
  verdict=$(echo "$record" | jq -r '.verdict // "missing"')
  remaining=$(echo "$record" | jq -r '.remaining // 0')

  if [[ "$verdict" == "pass" && "$remaining" -eq 0 ]]; then
    return 0
  fi

  echo "review gate failed — type=$review_type scope=$review_scope verdict=$verdict remaining=$remaining"
  return 1
}
```

Update `do_check_review` to call this helper. Note: capture the return code explicitly -- `$?` after a variable assignment reflects the subshell's exit code in bash, but capturing it separately is clearer and less error-prone:

```bash
do_check_review() {
  local plan_json="$1" review_type="$2" review_scope="$3"
  local plan_dir
  plan_dir="$(cd "$(dirname "$plan_json")" && pwd)"

  local result rc
  result=$(check_review_record "$plan_dir" "$review_type" "$review_scope")
  rc=$?
  if [[ $rc -ne 0 ]]; then
    echo "ERROR: $result" >&2
    exit 1
  fi
  exit 0
}
```

### Step 2: Gate `do_update_status_phase` on impl-review

Add a review gate at the top of `do_update_status_phase`, after the status validation but before the jq write. Only gate when the new status matches `Complete (YYYY-MM-DD)` — don't block transitions to "Not Started" or "In Progress":

```bash
if [[ "$new_status" =~ ^Complete ]]; then
  local plan_dir
  plan_dir="$(cd "$(dirname "$plan_json")" && pwd)"
  local phase_lower
  phase_lower=$(echo "$phase_letter" | tr '[:upper:]' '[:lower:]')

  local gate_result
  gate_result=$(check_review_record "$plan_dir" "impl-review" "phase-$phase_lower"); rc=$?
  if [[ $rc -ne 0 ]]; then
    echo "ERROR: cannot mark phase $phase_letter complete — $gate_result" >&2
    exit 1
  fi
fi
```

Note: scope uses lowercase `phase-a` not `phase-A` — matching the design doc convention where scope values use lowercase for consistency with directory naming.

### Step 3: Gate `do_update_status_plan` on all reviews

Add a review gate at the top of `do_update_status_plan`, when the new status is "Complete":

```bash
if [[ "$new_status" == "Complete" ]]; then
  local plan_dir
  plan_dir="$(cd "$(dirname "$plan_json")" && pwd)"
  local gate_errors=()

  local dr_result
  dr_result=$(check_review_record "$plan_dir" "design-review" "design"); rc=$?
  if [[ $rc -ne 0 ]]; then
    gate_errors+=("design-review: $dr_result")
  fi

  local pr_result
  pr_result=$(check_review_record "$plan_dir" "plan-review" "plan"); rc=$?
  if [[ $rc -ne 0 ]]; then
    gate_errors+=("plan-review: $pr_result")
  fi

  local json
  json=$(cat "$plan_json")
  local phase_count
  phase_count=$(echo "$json" | jq '.phases | length')

  for ((p=0; p<phase_count; p++)); do
    local letter
    letter=$(echo "$json" | jq -r ".phases[$p].letter")
    local letter_lower
    letter_lower=$(echo "$letter" | tr '[:upper:]' '[:lower:]')

    local ir_result
    ir_result=$(check_review_record "$plan_dir" "impl-review" "phase-$letter_lower"); rc=$?
    if [[ $rc -ne 0 ]]; then
      gate_errors+=("impl-review phase-$letter_lower: $ir_result")
    fi
  done

  if [[ "$phase_count" -gt 1 ]]; then
    local fr_result
    fr_result=$(check_review_record "$plan_dir" "impl-review" "final"); rc=$?
    if [[ $rc -ne 0 ]]; then
      gate_errors+=("impl-review final: $fr_result")
    fi
  fi

  if [[ ${#gate_errors[@]} -gt 0 ]]; then
    echo "ERROR: cannot mark plan complete — missing review gates:" >&2
    for ge in "${gate_errors[@]}"; do
      echo "  - $ge" >&2
    done
    exit 1
  fi
fi
```

### Step 4: Write test file tests/validate-plan/test_review_gates.sh

Create `tests/validate-plan/test_review_gates.sh` following the pattern from `tests/validate-plan/test_update_status.sh`. Test cases:

1. **Phase completion blocked without impl-review** -- set phase status to "In Progress", no reviews.json, attempt `--update-status --phase A --status "Complete (2026-03-23)"`, expect exit 1 with error about missing impl-review
2. **Phase completion succeeds with passing impl-review** -- add `[{"type":"impl-review","scope":"phase-a","verdict":"pass","remaining":0}]` to reviews.json, expect exit 0
3. **Phase "In Progress" transition not gated** -- no reviews.json, `--update-status --phase A --status "In Progress"`, expect exit 0 (only Complete is gated)
4. **Plan completion blocked without design-review** -- set all phases Complete with valid impl-reviews, no design-review record, attempt `--update-status --plan --status Complete`, expect exit 1
5. **Plan completion blocked without plan-review** -- add design-review but no plan-review, expect exit 1
6. **Plan completion succeeds with all reviews** -- add design-review, plan-review, impl-review for each phase, expect exit 0
7. **Multi-phase plan requires final impl-review** -- 2 phases with phase impl-reviews but no `scope:final` impl-review, expect exit 1
8. **Single-phase plan does not require final impl-review** -- 1 phase with phase impl-review, no final review, expect exit 0
9. **Run existing update-status tests** -- `bash tests/validate-plan/test_update_status.sh`
