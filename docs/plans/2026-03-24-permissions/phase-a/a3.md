**Avoid:** Don't create separate test files. All pretooluse hook tests belong in `tests/hooks/test_safe_commands.sh`.

**Avoid:** Don't hardcode safe-commands.txt entries in test files. Copy the bundled `hooks/safe-commands.txt` to `$TMPDIR_TEST` and use that path via `CLAUDE_SAFE_COMMANDS_FILE` so tests reflect actual production behavior.

**Avoid:** Don't test with `bash` in the safe list. The whole point of interpreter resolution is that `bash` is NOT safe — the hook resolves through it to the script basename.

## Steps

### Step 1: Add a deny-output assertion helper

Near the existing `assert_output_contains` and `assert_output_empty` helpers (around line 30), add:

```bash
assert_output_contains_deny_with_reason() {
  local desc="$1" output="$2" expected_reason="$3"
  if echo "$output" | grep -qF '"permissionDecision":"deny"' && echo "$output" | grep -qF "$expected_reason"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected deny with reason containing '$expected_reason')"
    echo "  Got: $output"
    ((FAIL++)) || true
  fi
}
```
