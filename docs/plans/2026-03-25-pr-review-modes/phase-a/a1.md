**Avoid:** Do not change `reviewer-prompt.md` — the design explicitly lists it as a non-goal. Do not exceed 2,000 words — SKILL.md files are injected into context and every excess word displaces working memory. Do not add new CLI arguments — the design says no new arguments beyond the existing set. Do not remove `--skip-review` or `--skip-fixes` — they remain valid in deliberate mode. Preserve the frontmatter description exactly (it controls skill triggering).

## Steps

### Step 1: Read the current SKILL.md and the design doc

Read `skills/pr-review/SKILL.md` and `docs/plans/2026-03-25-pr-review-modes/design-pr-review-modes.md` to understand both the current structure and target structure.

### Step 2: Rewrite SKILL.md

Replace the entire workflow section with the 9-step restructured workflow from the design. Keep the frontmatter, header, prerequisite line, and review principle unchanged.

The new workflow structure:

```markdown
### Step 1: Setup
```

Identical to the current Step 1 — detect PR, environment variables, checkout. No changes needed.

```markdown
### Step 2: Mode Selection
```

If `--automated`/`-A` flag is passed, skip prompt and use automated mode. Otherwise, AskUserQuestion with two options:
- **Automated** — Fix all actionable findings without interaction. External feedback processed first, then subagent findings.
- **Deliberate** — Collect all feedback, present unified triage, choose what to fix.

```markdown
### Step 3: Rebase onto Base Branch
```

Identical to the current Step 2 (rebase). No changes needed. Keep the existing bash snippet with `BASE_BRANCH`, the force-push note about outdated bot comments.

```markdown
### Step 4: Dispatch Subagent in Background
```

Both modes dispatch the reviewer subagent with `run_in_background: true`. Reference `reviewer-prompt.md` using `**See:** reviewer-prompt.md` syntax. Template variables: `{DIFF_RANGE}` = `origin/$BASE_BRANCH..HEAD`, `{REPO_PATH}` = repo root, `{PR_NUMBER}` = PR number. Skip if `--skip-review` was passed.

Key change from current Step 3: the subagent always runs in background (previously it was synchronous).

```markdown
### Step 5: External Feedback
```

Three paths based on mode:

**`--automated` flag (from orchestrate):** Skip warm-up and polling — orchestrate already polled checks. Collect available feedback via `gh`. Assess and categorize (same categories as current Step 4: actionable fix, suggestion, informational, false positive). Fix all actionable items. Run tests. `git commit` locally (do NOT push — wave 2 may modify same files).

**User-selected automated:** Wait 60 seconds for bots to register checks. Poll `gh pr checks $PR_NUMBER` every 60 seconds. Scan latest PR comments for "processing" / "in progress" indicators. Ready when: all checks complete AND no processing indicators. CodeRabbit rate-limit warning = proceed immediately. Timeout: 10 minutes. Collect, assess, fix all actionable, run tests, `git commit` locally.

**Deliberate:** No warm-up. Poll `gh pr checks` every 60 seconds with same readiness signals (all checks complete, no processing indicators, rate-limit = proceed, 10 min timeout). Collect external feedback and report status. No fixes yet — wait for unified triage.

```markdown
### Step 6: Subagent Results
```

Wait for background subagent to return.

**Automated:** Assess findings against current working tree (post-wave-1 fixes). Dismiss findings already addressed by wave-1. Fix remaining actionable items. Run tests. `git commit` and `git push -u origin HEAD` (single push covers both waves).

**Deliberate:** Collect subagent findings. Merge with external feedback into unified finding set. Proceed to Step 7.

```markdown
### Step 7: Present & Confirm (Deliberate Only)
```

Same as current Step 5 — show summary table with source, category, planned action. AskUserQuestion with options: Fix all, Fix critical only, Skip fixes, Other. Omit "Skip fixes" when `--automated` (but this step is deliberate-only, so that case doesn't apply).

```markdown
### Step 8: Fix, Test, Push (Deliberate Only)
```

Same as current Step 6. For each actionable item: make fix, run tests, commit and push with `git push -u origin HEAD`.

```markdown
### Step 9: Comment on PR
```

Same as current Step 7. Post `gh pr comment` with unified assessment. Automated mode skips merge prompt (caller handles). Deliberate mode offers merge prompt via AskUserQuestion (Merge PR / Not yet).

**Arguments table:** Keep existing arguments unchanged:

- `--skip-fixes` / `-S` — skip fixing (invalid with `--automated`)
- `--automated` / `-A` — force fixes, suppress interaction

**Pitfalls table:** Keep the existing pitfalls plus add:
