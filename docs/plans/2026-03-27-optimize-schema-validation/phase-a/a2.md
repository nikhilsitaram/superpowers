### Step 2: Bulk extract all status data with a single jq call

Replace the ~20 individual jq calls with a single extraction that produces a compact representation of all plan/phase/task status relationships:

```bash
local _cdata
_cdata=$(echo "$json" | jq -c '{
  plan_status: (.status // ""),
  phases: [.phases // [] | to_entries[] | {
    idx: .key,
    letter: (.value.letter // ""),
    status: (.value.status // ""),
    tasks: [(.value.tasks // []) | to_entries[] | {
      idx: .key,
      id: (.value.id // ""),
      status: (.value.status // ""),
      depends_on: (.value.depends_on // [])
    }]
  }]
}')
```

This is 1 jq call replacing ~20.

### Step 5: Optimize Rule 3 — dependency completion check

Rule 3 (lines 508-536) has an O(n^2) inner loop that scans all phases/tasks to find each dependency's status. With the extracted `_cdata`, build a bash associative array mapping task ID to status:

```bash
declare -A _task_statuses
for ((p=0; p<phase_count; p++)); do
  local _tc
  _tc=$(echo "$_cdata" | jq ".phases[$p].tasks | length")
  for ((t=0; t<_tc; t++)); do
    local _tid _ts
    _tid=$(echo "$_cdata" | jq -r ".phases[$p].tasks[$t].id")
    _ts=$(echo "$_cdata" | jq -r ".phases[$p].tasks[$t].status")
    _task_statuses["$_tid"]="$_ts"
  done
done
```

Then the dependency check becomes O(1) lookup:

```bash
if [[ "$ts" == "complete" ]]; then
  local _deps
  _deps=$(echo "$_task" | jq -r '.depends_on[]')
  while IFS= read -r dep_id; do
    [[ -z "$dep_id" ]] && continue
    local dep_status="${_task_statuses[$dep_id]:-}"
    if [[ "$dep_status" == "pending" || "$dep_status" == "in_progress" ]]; then
      err "ERROR: status_inconsistency: task $tid is 'complete' but dependency $dep_id is '$dep_status'"
    fi
  done <<< "$_deps"
fi
```

**Even better:** Build the associative array in a single pass during the initial phase iteration (Step 4), avoiding the separate loop.
