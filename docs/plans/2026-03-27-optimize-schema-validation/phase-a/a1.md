## Extraction Strategy

Use NUL-delimited (`\u0000`) jq output for safe parsing of multi-word fields. TSV breaks on fields like `status: "Not Yet Started"` that contain spaces. The pattern:

```bash
local _vals
_vals=$(echo "$json" | jq -j '[ field1, field2, ... ] | map(. + "\u0000") | join("")')

local _arr=()
while IFS= read -r -d '' val; do
  _arr+=("$val")
done <<< "$_vals"
```

This is 1 jq fork per extraction level, with zero forks for field access.

For nested structures (depends_on arrays, files objects), extract as compact JSON-per-line and use jq on the small per-entity JSON (~100-200 bytes, ~1ms each). The goal is eliminating full-plan jq calls in loops, not reaching zero jq forks.
