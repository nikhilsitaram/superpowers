Then extract each field from the small JSON object using `jq -r` on `top_json` (this is fast because the input is tiny ~200 bytes, not the full plan). Even better, use a single jq with `@tsv`:

```bash
local schema status goal architecture tech_stack workflow execution_mode ib_exists ib_type integration_branch plan_criteria_count phase_count
read -r schema status goal architecture tech_stack workflow execution_mode ib_exists ib_type integration_branch plan_criteria_count phase_count < <(

    (.schema // "" | tostring),
    (.status // ""),

```

**Important TSV caveat:** Fields containing tabs or multi-word values break TSV parsing. Since `status` can be "Not Yet Started" (contains spaces) and `goal`/`architecture`/`tech_stack` contain arbitrary text with spaces, TSV with `read -r` will merge remaining fields into the last variable.

**Use JSON-per-line instead for safety.** Extract into a compact JSON object, then use a single secondary jq call on the tiny result:

```bash
local _top

  schema: (.schema // "" | tostring),
  status: (.status // ""),
  goal: (.goal // ""),
  architecture: (.architecture // ""),
  tech_stack: (.tech_stack // ""),
  workflow: (.workflow // ""),
  execution_mode: (.execution_mode // ""),
  ib_exists: has("integration_branch"),
  ib_type: (.integration_branch | type),
  integration_branch: (if has("integration_branch") then (.integration_branch // "") | tostring else "" end),
  plan_criteria_count: ((.success_criteria // []) | length),
  phase_count: ((.phases // []) | length)
}')

schema=$(echo "$_top" | jq -r '.schema')
status=$(echo "$_top" | jq -r '.status')
goal=$(echo "$_top" | jq -r '.goal')
architecture=$(echo "$_top" | jq -r '.architecture')
tech_stack=$(echo "$_top" | jq -r '.tech_stack')
workflow=$(echo "$_top" | jq -r '.workflow')
execution_mode=$(echo "$_top" | jq -r '.execution_mode')
local ib_exists ib_type integration_branch plan_criteria_count phase_count

ib_type=$(echo "$_top" | jq -r '.ib_type')

```

**Wait — this is still 13 jq calls on the small object.** The point is to avoid forking jq at all for field extraction from the tiny object. Use bash-native JSON parsing with a NUL-delimited approach instead:

```bash
local _vals
_vals=$(echo "$json" | jq -j '[
  (.schema // "" | tostring),
  (.status // ""),
  (.goal // ""),
  (.architecture // ""),
  (.tech_stack // ""),
  (.workflow // ""),
  (.execution_mode // ""),
  (has("integration_branch") | tostring),
  (.integration_branch | type),
  (if has("integration_branch") then (.integration_branch // "") | tostring else "" end),
  ((.success_criteria // []) | length | tostring),
  ((.phases // []) | length | tostring)
] | map(. + "\u0000") | join("")')

local _arr=()
while IFS= read -r -d '' val; do
  _arr+=("$val")
done <<< "$_vals"

schema="${_arr[0]}"
status="${_arr[1]}"
goal="${_arr[2]}"
architecture="${_arr[3]}"
tech_stack="${_arr[4]}"
workflow="${_arr[5]}"
execution_mode="${_arr[6]}"
local ib_exists="${_arr[7]}" ib_type="${_arr[8]}" integration_branch="${_arr[9]}"
local plan_criteria_count="${_arr[10]}" phase_count="${_arr[11]}"
```

This is **1 jq fork** replacing the original ~10. The NUL delimiter handles fields with spaces, newlines, and tabs correctly.

Then keep the existing validation logic (the `[[ -z "$schema" ]]` checks, case statements, etc.) exactly as-is.

### Step 2: Bulk extract plan-level success_criteria (replaces ~3 jq calls per criterion)

