## Steps

### Step 1: Replace the branch creation model

Replace the current phase branch creation:

```text

```

With per-phase worktrees from the integration branch:

```text
git worktree add .claude/worktrees/<feature>-phase-{letter} -b phase-{letter} integrate/<feature>
```

The integration branch is already set up by the design skill. The orchestrator creates phase worktrees as siblings of the integration worktree.

### Step 2: Update PR targeting

Replace stacked PR base (`--base phase-{prior-letter}` / `--base main`) with integration branch base:

```text
Ship PR: invoke ship with --base integrate/<feature>
```

All phase PRs target the integration branch. After each phase PR merges (via `gh pr merge`), update the integration worktree with `git pull`.

### Step 3: Add workflow routing

At the start of execution, read the workflow field from plan.json:

```bash

```

At the end (after all phases complete):
- `"ship"`: create + merge final PR from `integrate/<feature>` → main, clean up worktrees
- `"review-only"`: create final PR but stop — user reviews and merges manually

Change from "phases before the current one (indices 0..N-1)" to "transitive closure of depends_on chain." For Phase A this stays empty. For later phases, concatenate completion.md files only from phases in the dependency chain.

Note: in this phase (sequential dispatch), this is functionally the same as "all prior phases" since phases run in order. But the language must be correct for Phase B when parallel dispatch is added.

### Step 5: Add phase worktree cleanup

After each phase PR merges, clean up:

```bash

```
