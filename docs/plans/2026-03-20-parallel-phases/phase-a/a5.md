# A5: Add --base argument to ship SKILL.md

**Avoid:** Don't change ship's default behavior — when `--base` is omitted, ship should continue targeting `$DEFAULT_BRANCH` as it does today. The `--base` argument is only used when orchestrate explicitly passes it for phase PRs.

## Steps

### Step 1: Add --base to the Arguments table

In the Arguments table at the bottom of ship SKILL.md, add:

| `--base <branch>` | Target specific base branch for PR (default: `$DEFAULT_BRANCH`) |

### Step 2: Update Step 8 (Create PR) to use --base

In Step 8, modify the `gh pr create` command to include `--base` when provided:

```bash
BASE_FLAG=""
if [ -n "$BASE_BRANCH" ]; then BASE_FLAG="--base $BASE_BRANCH"; fi
gh pr create $BASE_FLAG --title "..." --body "..."
```

Add a note: "When `--base` is provided (e.g., from orchestrate for phase PRs), the PR targets that branch instead of the default. This enables the integration branch model where phase PRs target `integrate/<feature>`."

### Step 3: Verify word count and commit

Count words: `wc -w skills/ship/SKILL.md` — must be under 1000. Commit.
