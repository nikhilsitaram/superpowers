# A4: Update design SKILL.md for integration branch and workflow routing

**Avoid:** Don't add the full parallel dispatch logic — that's orchestrate's job. The design skill only needs to (1) create the integration branch worktree, (2) ask the user about workflow preference, and (3) write the choice to plan.json. Keep it under 1000 words.

## Steps

### Step 1: Update worktree setup step

Change step 7 from:

```text
git worktree add .worktrees/<branch-name> -b <branch-name>
```

To:

```text
git worktree add .claude/worktrees/<feature> -b integrate/<feature>
```

Where `<feature>` is the kebab-case topic name (e.g., `parallel-phases`). Add a note that this is the integration branch — phase worktrees are created by orchestrate as siblings.

### Step 2: Add workflow routing step

After step 10 (dispatch draft-plan), add a new step 11:

**Route workflow** — After draft-plan returns, ask the user:

AskUserQuestion (header: "Workflow"):
- **Ship** — Full auto: orchestrate executes, reviews, and ships
- **Review only** — Orchestrate executes and reviews, creates final PR but doesn't merge
- **Plan only** — Stop here. Plan is written and reviewed.

Write the chosen value to plan.json: `jq --arg w "<value>" '.workflow = $w' plan.json > tmp && mv tmp plan.json`

For **Ship** or **Review only**: invoke orchestrate.
For **Plan only**: report the plan file path and stop.

### Step 3: Update the Agent dispatch examples

Update the two `Agent()` blocks in the Checklist section to use `.claude/worktrees/<feature>` as the working directory path instead of `<absolute-worktree-path>`.

### Step 4: Verify word count and commit

Count words: `wc -w skills/design/SKILL.md` — must be under 1000. If over, trim existing prose (the Challenging Assumptions and Smart Question Batching sections have room for compression). Commit.
