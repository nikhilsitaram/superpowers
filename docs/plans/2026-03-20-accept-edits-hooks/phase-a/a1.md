# A1: Write test suites for both hooks

**Avoid:** Don't test by calling the hook scripts directly with `bash hooks/script.sh` — the hooks read from stdin and write to stdout/files, so each test must pipe JSON to the script and check side effects. Follow the same test harness pattern as `tests/validate-plan/test_schema.sh` (assert_pass/assert_fail helpers, PASS/FAIL counters, temp directories with trap cleanup).

**Avoid:** Don't hardcode absolute paths in test fixtures. The tests create temp directories to simulate worktree structures and plan directories, so all paths are derived from `mktemp -d`.

## Steps

### Step 1: Create test directory and PostToolUse test file

Create `tests/hooks/test_post_tool_use.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
HOOK="$REPO_ROOT/hooks/post-tool-use-design-approval.sh"
PASS=0
FAIL=0

assert_file_exists() {
  local desc="$1" file="$2"
  if [[ -f "$file" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (file not found: $file)"
    ((FAIL++)) || true
  fi
}

assert_file_not_exists() {
  local desc="$1" file="$2"
  if [[ ! -f "$file" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (file should not exist: $file)"
    ((FAIL++)) || true
  fi
}

assert_file_contains() {
  local desc="$1" file="$2" expected="$3"
  if [[ -f "$file" ]] && grep -qF "$expected" "$file"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected' in $file)"
    ((FAIL++)) || true
  fi
}

TMPDIR=$(mktemp -d)
trap 'rm -rf "$TMPDIR"' EXIT
```

Then add the following test cases (each builds a JSON payload, pipes it to the hook, and checks the `.design-approved` sentinel file):

**Test 1: Approved with metadata.source creates sentinel**
Build stdin JSON with `tool_name: "AskUserQuestion"`, `tool_input.metadata.source: "design-approval"`, `tool_input.questions[0].question` containing `"Plan dir: $TMPDIR/docs/plans/2026-03-20-topic"`, `tool_response: "Approved"`, `session_id: "test-session-123"`. Pipe to hook. Assert `$TMPDIR/docs/plans/2026-03-20-topic/.design-approved` exists and contains `test-session-123`.

**Test 2: Approved with text fallback creates sentinel (no metadata)**
Same as Test 1 but omit `metadata` field from `tool_input`. The hook should detect via the `"Plan dir:"` prefix in question text. Assert sentinel created.

**Test 3: Rejected ("Needs changes") does not create sentinel**
Same as Test 1 but `tool_response: "Needs changes"`. Assert sentinel file does NOT exist.

**Test 4: Non-AskUserQuestion tool is ignored**
Set `tool_name: "Edit"`. Assert sentinel file does NOT exist.

**Test 5: AskUserQuestion without design-approval metadata or Plan dir text is ignored**
Set `tool_name: "AskUserQuestion"` with a question that does NOT contain `"Plan dir:"` and no metadata.source. Assert sentinel file does NOT exist.

**Test 6: session_id written correctly to sentinel**
Reuse Test 1 with a different session_id value (`"session-abc-456"`). Assert file contents match exactly.

**Test 7: mkdir -p creates intermediate directories**
Use a plan dir path where the parent directories don't exist yet (`$TMPDIR/new/nested/docs/plans/2026-03-20-test`). Assert sentinel created at the deep path.

End with summary:

```bash
echo ""
echo "$PASS passed, $FAIL failed"
[[ $FAIL -eq 0 ]]
```

### Step 2: Create PermissionRequest test file

Create `tests/hooks/test_permission_request.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
HOOK="$REPO_ROOT/hooks/permission-request-accept-edits.sh"
PASS=0
FAIL=0

assert_output_contains() {
  local desc="$1" output="$2" expected="$3"
  if echo "$output" | grep -qF "$expected"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected' in output)"
    ((FAIL++)) || true
  fi
}

assert_output_empty() {
  local desc="$1" output="$2"
  if [[ -z "$output" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected empty output, got: $output)"
    ((FAIL++)) || true
  fi
}

TMPDIR=$(mktemp -d)
trap 'rm -rf "$TMPDIR"' EXIT
```

Then add the following test cases (each sets up a directory structure with or without a sentinel, pipes JSON to the hook, captures stdout):

**Test 1: Matching sentinel returns allow+setMode JSON**
Create `$TMPDIR/docs/plans/2026-03-20-topic/.design-approved` containing `"sess-001"`. Build stdin JSON with `session_id: "sess-001"`, `cwd: "$TMPDIR"`. Pipe to hook. Assert stdout contains `"behavior": "allow"` and `"mode": "acceptEdits"` and `"destination": "session"`.

**Test 2: Mismatched session_id produces no output (passthrough)**
Create sentinel containing `"sess-old"`. Build stdin JSON with `session_id: "sess-new"`, `cwd: "$TMPDIR"`. Assert stdout is empty (exit 0, no JSON output).

**Test 3: No sentinel file produces no output (passthrough)**
Build stdin JSON with `session_id: "sess-001"`, `cwd: "$TMPDIR"` but create NO `.design-approved` file. Assert stdout is empty.

**Test 4: Worktree search path finds sentinel**
Create `$TMPDIR/.claude/worktrees/my-branch/docs/plans/2026-03-20-topic/.design-approved` containing `"sess-wt"`. Build stdin JSON with `session_id: "sess-wt"`, `cwd: "$TMPDIR"`. Assert stdout contains allow+setMode JSON.

**Test 5: Direct cwd path works (cwd IS the worktree)**
Create `$TMPDIR/docs/plans/2026-03-20-topic/.design-approved` containing `"sess-direct"`. Build stdin JSON with `session_id: "sess-direct"`, `cwd: "$TMPDIR"`. Assert stdout contains allow+setMode JSON.

**Test 6: Multiple sentinels, only matching session_id triggers**
Create two sentinels: one with `"sess-old"` and one with `"sess-current"` in different plan dirs. Build stdin JSON with `session_id: "sess-current"`. Assert stdout contains allow JSON.

End with summary:

```bash
echo ""
echo "$PASS passed, $FAIL failed"
[[ $FAIL -eq 0 ]]
```
