# A2: Write consistency mode tests

**Avoid:** Don't hardcode plan.json structures inline when you can derive them from the fixture. Use `jq` mutations on the valid-plan fixture to create test scenarios — this keeps tests resilient to fixture changes.

**Avoid:** Don't test via `--schema` for the new rules. These tests should call `--consistency` directly to verify the standalone mode works. The existing `test_status_consistency.sh` already covers the `--schema` chaining path for Rules 2 and 5.

## Steps

### Step 1: Create test file with boilerplate

Create `tests/validate-plan/test_consistency.sh` with the standard test harness used by other test files:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
VALIDATE="$REPO_ROOT/scripts/validate-plan"
FIXTURES="$SCRIPT_DIR/fixtures"
PASS=0
FAIL=0

assert_pass() {
  local desc="$1"; shift
  if "$@" > /dev/null 2>&1; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc"
    ((FAIL++)) || true
  fi
}

assert_fail() {
  local desc="$1"; shift
  local expected_error="$1"; shift
  local output
  if output=$("$@" 2>&1); then
    echo "FAIL: $desc (expected failure, got success)"
    ((FAIL++)) || true
  elif echo "$output" | grep -q "$expected_error"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected_error' in output, got: $output)"
    ((FAIL++)) || true
  fi
}

setup_valid_plan() {
  local dir="$1"
  rm -rf "${dir:?}/"*
  cp -r "$FIXTURES/valid-plan/"* "$dir/"
  cp "$FIXTURES/valid-plan/plan.json" "$dir/plan.json"
}

TMPDIR=$(mktemp -d)
trap 'rm -rf "$TMPDIR"' EXIT
```

### Step 2: Write test cases for all 6 rules

Add tests for each consistency rule, both positive (consistent state passes) and negative (inconsistent state fails with `status_inconsistency` error):

**Rule 1 — Phase "Not Started" but has active tasks:**

```bash
echo "Test 1: Rule 1 — Phase Not Started with in_progress task fails"
setup_valid_plan "$TMPDIR"
jq '.phases[0].status = "Not Started" | .phases[0].tasks[0].status = "in_progress"' "$TMPDIR/plan.json" > "$TMPDIR/tmp.json" && mv "$TMPDIR/tmp.json" "$TMPDIR/plan.json"
assert_fail "phase not-started with in_progress task" "status_inconsistency" \
  "$VALIDATE" --consistency "$TMPDIR/plan.json"

echo "Test 2: Rule 1 — Phase Not Started with complete task fails"
setup_valid_plan "$TMPDIR"
jq '.phases[0].status = "Not Started" | .phases[0].tasks[0].status = "complete"' "$TMPDIR/plan.json" > "$TMPDIR/tmp.json" && mv "$TMPDIR/tmp.json" "$TMPDIR/plan.json"
assert_fail "phase not-started with complete task" "status_inconsistency" \
  "$VALIDATE" --consistency "$TMPDIR/plan.json"

echo "Test 3: Rule 1 — Phase Not Started with all pending tasks passes"
setup_valid_plan "$TMPDIR"
assert_pass "phase not-started with all pending tasks" \
  "$VALIDATE" --consistency "$TMPDIR/plan.json"
```
