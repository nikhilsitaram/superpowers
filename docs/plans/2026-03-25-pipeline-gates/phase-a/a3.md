# A3: Write check-entry mode tests

**Avoid:** Don't create a real plan.json for the `draft-plan` stage tests. The design doc specifies that `--check-entry --stage draft-plan` only reads `reviews.json` from the plan directory — plan.json need not exist. However, the CLI still requires a plan.json path argument for interface consistency. Create a dummy plan.json that the command won't actually parse, or use the fixture.

## Steps

### Step 1: Create test file with boilerplate

Create `tests/validate-plan/test_check_entry.sh` with the standard test harness (same pattern as other test files — `assert_pass`, `assert_fail`, `setup_valid_plan`, `TMPDIR` with cleanup trap).

### Step 2: Write draft-plan stage tests

```bash
echo "Test 1: draft-plan stage — design-review passed, exits 0"
setup_valid_plan "$TMPDIR"
printf '[{"type":"design-review","scope":"design","verdict":"pass","remaining":0}]' > "$TMPDIR/reviews.json"
assert_pass "draft-plan with design-review passed" \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json" --stage draft-plan

echo "Test 2: draft-plan stage — no design-review, exits 1"
setup_valid_plan "$TMPDIR"
echo '[]' > "$TMPDIR/reviews.json"
assert_fail "draft-plan without design-review" "entry gate failed" \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json" --stage draft-plan

echo "Test 3: draft-plan stage — design-review failed, exits 1"
setup_valid_plan "$TMPDIR"
printf '[{"type":"design-review","scope":"design","verdict":"fail","remaining":2}]' > "$TMPDIR/reviews.json"
assert_fail "draft-plan with failed design-review" "entry gate failed" \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json" --stage draft-plan

echo "Test 4: draft-plan stage — no reviews.json, exits 1"
setup_valid_plan "$TMPDIR"
rm -f "$TMPDIR/reviews.json"
assert_fail "draft-plan without reviews.json" "entry gate failed" \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json" --stage draft-plan

echo "Test 5: draft-plan stage — plan.json doesn't exist but reviews.json does, exits 0"
rm -rf "${TMPDIR:?}/"*
mkdir -p "$TMPDIR"
printf '[{"type":"design-review","scope":"design","verdict":"pass","remaining":0}]' > "$TMPDIR/reviews.json"
assert_pass "draft-plan without plan.json but with reviews.json" \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json" --stage draft-plan
```

This last test is critical: it verifies the design doc requirement that plan.json doesn't need to exist for the `draft-plan` stage. The `--check-entry` function derives the directory from the plan.json path but only reads reviews.json.

### Step 4: Write edge case tests

Use the `assert_exit` helper (from `test_check_review.sh` pattern) for exit code tests. Add the helper to the boilerplate:

```bash
assert_exit() {
  local desc="$1" expected_exit="$2"; shift 2
  local actual_exit=0
  "$@" > /dev/null 2>&1 || actual_exit=$?
  if [[ "$actual_exit" -eq "$expected_exit" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected exit $expected_exit, got $actual_exit)"
    ((FAIL++)) || true
  fi
}
```

Then use it for edge case tests:

```bash
echo "Test 10: unknown stage exits 2"
setup_valid_plan "$TMPDIR"
printf '[{"type":"design-review","scope":"design","verdict":"pass","remaining":0}]' > "$TMPDIR/reviews.json"
assert_exit "unknown stage exits 2" 2 \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json" --stage bogus

echo "Test 11: missing --stage flag exits 2"
setup_valid_plan "$TMPDIR"
assert_exit "missing --stage exits 2" 2 \
  "$VALIDATE" --check-entry "$TMPDIR/plan.json"
```
