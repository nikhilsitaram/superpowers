# A7: Update design skill doc to write integration_branch

**Avoid:** Don't add integration_branch writing to every worktree setup path. Only multi-phase plans get an integration branch. The design doc specifies: "Set by the design skill when creating an integration worktree for multi-phase plans."

## Steps

### Step 1: Add integration_branch write to the worktree setup step in `skills/design/SKILL.md`

In the Checklist section, step 6 covers worktree setup. The multi-phase bullet currently reads:

```text
- Multi-phase: `git worktree add -b integrate/<feature> .claude/worktrees/<feature>` — integration branch; phase worktrees created by orchestrate as siblings
```

After this line, add instruction to write the integration branch name to plan.json when it's created later. The most natural place is in step 11 (Route workflow) where the design skill already writes `workflow` and `execution_mode` to plan.json. Add to that jq command:

In step 11, after the existing jq command that writes workflow and execution_mode, add a conditional for multi-phase plans:

```markdown
For multi-phase plans, also write the integration branch name:
`jq --arg ib "integrate/<feature>" '.integration_branch = $ib' plan.json > tmp && mv tmp plan.json`
```

Alternatively, fold it into the existing jq pipeline in step 11:

```markdown
(Multi-phase only) Write integration_branch: `jq --arg w "<workflow>" --arg e "<exec-mode>" --arg ib "integrate/<feature>" '.workflow = $w | .execution_mode = $e | .integration_branch = $ib' plan.json > tmp && mv tmp plan.json`
```

The key is that `integration_branch` gets written at the same time as the other plan.json overrides, and only for multi-phase plans.

### Step 2: Verify

```bash
grep -q 'integration_branch' skills/design/SKILL.md
```

Commit: `docs: add integration_branch write to design skill doc`
