# A8: Bump version in marketplace.json

**Avoid:** Don't hardcode the target version number. Read the current version from `.claude-plugin/marketplace.json` at implementation time and increment the minor version. The worktree may be behind main, so the current version might differ from what the design doc assumed.

## Steps

### Step 1: Read current version

```bash
jq -r '.plugins[0].version' .claude-plugin/marketplace.json
```

Note the current version.

### Step 2: Bump all three plugin versions

Increment the minor version (e.g., 1.4.0 -> 1.5.0, or 1.8.0 -> 1.9.0). Update all three plugins to the same new version in `.claude-plugin/marketplace.json`.

### Step 3: Run all hook tests to verify nothing broke

```bash
bash tests/hooks/test_safe_commands.sh
bash tests/hooks/test_post_tool_use.sh
bash tests/hooks/test_permission_request.sh
```

All three test suites should pass. This verifies SC5 (existing pipeline behavior unchanged).

### Step 4: Verify version bump

```bash
jq -r '.plugins[].version' .claude-plugin/marketplace.json | sort -u
```

Should output a single version number (all three match). Commit:

```text
chore: bump plugin version for safe commands hook feature
```
